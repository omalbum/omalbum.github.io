Authentication

POST /auth/login

Requires auth

False

Description

Request for login. Returns 401 if fails, otherwise 200.

Returns the token with the expiration date. Once authenticated, the next protected requests must be done with the following header:

The field user_name can be the user name or the email.

Authorization: Bearer TOKEN

Payload to send

{

        "user_name": "jperez",

        "password": "secreto123",

}

Example response

{

    "token": "eyJhbGciOasdasda.sasdasdasdgFDN-XNJBasdasdasdC-g",

    "expiration": "2020-06-28T12:00:37-03:00",

    "User": {

        "user_id": 2,

        "user_name": "juan",

        "name": "Juan",

        "last_name": "Perez",

        "email": "juan.perez@gmail.com"

    }

}

PUT /admin/problem/:problem_id

Requires auth

True

Description

An admin can use this endpoint to edit a problem.

If the problem has been released it is not possible to change the release date.

The series can only be changed for drafts.

Returns 200 or 404 if problem is not found or 400 if validation fails

Payload to send

{

        "statement": "Sea $n$ entero entonces $n^2>0$.",

        "answer": 42,

        "omaforos_post_id" : 1234,

        "annotations" : "",

        "hint": "es facil!",

        "tags" : ["nivel1","algebra"],

        "official_solution" : "te la debo",

        "release_date": "2020-08-01T21:00:00-03:00",

        "deadline": "2020-08-02T21:00:00-03:00",

        "is_draft": false

True

Description

A user may attempt a problem once during the contest. The result of the attempt is not returned during contest.
